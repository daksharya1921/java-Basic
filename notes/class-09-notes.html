🧠 Key Concepts

Assignment Operator (=)

Used to set/assign a value to a variable. It is not mathematical equality. Example: int amount = 200; means "set amount to 200", not "amount equals 200".

Relational Operators

Used to compare values and check relationships. They always evaluate to true or false (Java has no concept of "yes/no").

  • > greater than
  • < less than
  • == equal to (comparison, not assignment)
  • != not equal to
  • >= greater than or equal to
  • <= less than or equal to

The result of a relational comparison is called a condition.

Logical Operators — AND (&&)

Represented by double ampersand (&&). Both conditions must be true for the overall result to be true. If the first condition is false, the second condition is not even checked — the overall result is immediately false. This is called short-circuiting.

Logical Operators — OR (||)

Represented by double pipe (||). If either condition is true, the overall result is true. If the first condition is already true, the second condition is not checked — this is the English "or" (not "both"), unlike Hindi "aur" which implies both.

if-else Statement

Used to execute a block of code only when a condition is true:

if (condition) {
    // executes if condition is true
} else {
    // executes if condition is false
}

Only one of the two blocks executes per run — never both.

Comparing Strings — Use .equals(), Not ==

To compare whether two String values are equal, use the .equals() method (a method inside the String class), e.g. state.equals("Karnataka"). This returns true or false based on whether the string content matches.

Real-World Use of Relational/Logical Operators

Business rules are typically implemented using these operators — e.g. minimum order quantity checks, free delivery thresholds (e.g. "free delivery above ₹450"), discount eligibility (e.g. "10% off above ₹800"), and age eligibility for a driving license (18–60 years).

Reading and Converting Command Line Input

Command line arguments are read as Strings and converted to int using Integer.parseInt() before being used in numeric/relational comparisons.

DL eligibility check using AND:

class DLEligibility {

    public static void main(String[] args) {
        int minAge = 18;
        int maxAge = 60;

        String ageStr = args[0];
        int applicantAge = Integer.parseInt(ageStr);

        if (applicantAge >= minAge && applicantAge <= maxAge) {
            System.out.println("Allowed to apply for DL");
        } else {
            System.out.println("You are not allowed to apply for DL, your age is " + applicantAge);
        }
    }

}

Admission eligibility using OR and String comparison:

class AdmissionCheck {

    public static void main(String[] args) {
        String state = args[0];
        int marks = Integer.parseInt(args[1]);

        if (marks >= 75 || state.equals("Karnataka")) {
            System.out.println("Allowed to take admission");
        } else {
            System.out.println("Not allowed to take admission");
        }

        System.out.println("Marks: " + marks);
        System.out.println("State: " + state);
    }

}

Delivery charges & discount calculator (practice exercise):

class OrderCharges {

    public static void main(String[] args) {
        String customerType = args[0]; // "premium" or "non-premium"
        int price = Integer.parseInt(args[1]);

        int deliveryCharge = 0;
        int discount = 0;

        if (customerType.equals("premium") || price >= 5000) {
            deliveryCharge = 0; // free delivery
            discount = price * 10 / 100; // 10% discount
        } else {
            deliveryCharge = 30; // flat delivery charge
        }

        int totalCartValue = price - discount + deliveryCharge;

        System.out.println("Customer Type: " + customerType);
        System.out.println("Price: " + price);
        System.out.println("Delivery Charge: " + deliveryCharge);
        System.out.println("Discount: " + discount);
        System.out.println("Total Cart Value: " + totalCartValue);
    }

}

📌 Important Points / Common Mistakes

  • Never confuse = (assignment) with == (comparison) — this is a very common beginner mistake.
  • Relational and logical operators always return true or false, never "yes/no".
  • AND requires both conditions true; OR requires at least one condition true.
  • Command line arguments are strings — convert to int using Integer.parseInt() before doing numeric comparisons.
  • Use String's .equals() method to compare string values, not direct equality symbols.
  • Only one branch of an if-else executes per run — never both.
  • Keep business logic simple and match it precisely to the stated requirement (many students in class made logic errors mixing up premium/non-premium delivery and discount rules).
