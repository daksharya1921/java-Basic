🧠 Key Concepts

What Access Modifiers Control

Access modifiers decide who can call or use a class, method, variable, or constructor — specifically, which other classes are allowed to access them. This gives a developer control over what should be exposed and what should stay hidden/protected within the code.

private — Scope: Within the Declaring Class Only

A private member (method, variable, or constructor) can only be accessed within the same class where it is declared. Any other class — even in the same file — trying to call a private member gets a compiler error: has private access in [ClassName].

Why Packages Exist

In large systems (e.g. Amazon-scale applications), thousands of developers may independently create classes with the same name (e.g. User.java, Account.java) for different purposes (retail user, reseller user, business user). Since two files with the same name cannot exist in the same folder, Java uses packages — a folder/namespace structure — to organize classes and allow classes with the same name to coexist in different packages.

Package = Folder Structure / Namespace

A package is essentially a folder that organizes related classes (this is called being modular — like organizing a house into a kitchen, bedroom, and living room instead of keeping everything in one open space). To uniquely identify a class, you combine its package name + class name — similar to using a person's name plus their father's name to distinguish between two people with the same first name.

Package Naming Convention

By convention, a package name is the reverse of the company's domain name. Example: for a company with domain codewala.com, the package would start with com.codewala.

Declaring and Using Packages

The first line of a Java file (when using packages) is the package declaration:

package com.codewala;

The file must physically reside inside a matching folder structure (e.g. com/codewala/Account.java). Compilation and execution commands must reference the fully qualified path/package.

default (No Modifier) — Scope: Within the Same Package

If no access modifier is written, the member gets default access. A default member can be accessed by any other class within the same package, but not from a class in a different package — even if that class imports it, compilation fails with an error like: is not public in [package]; cannot be accessed from outside package.

protected — Scope: Same Package + Child Classes in Other Packages

Protected has the same visibility as default (same package), plus it can be accessed from a class in a different package if that class is a child (subclass) of the class where the member is declared. Full understanding of this requires inheritance, which is an upcoming topic.

public — Scope: Accessible from Anywhere

A public member can be accessed from any class, in any package — the widest possible visibility.

Access Modifier Visibility Ranking (Narrowest to Widest)
🔒 private
class only
📦 default
package only
🌐 protected
package + child classes
🌍 public
everywhere
Importing Classes from Another Package

To use a class from a different package, it must be imported using an import statement — similar to importing a product from another country before it can be used domestically.

import com.codewala.Account;
Class Access Modifier vs Member Access Modifier

A class itself can be marked public or default. Even if a class is public (callable from outside its package), the individual methods/members inside it still follow their own access modifiers — a private method inside a public class still cannot be called from outside that class.

💻 Code Snippets / Examples

private — accessible only within the same class:

class Account {

    public static void main(String[] args) {
        Account.accountInfo(); // works — called within the same class
    }

    private static void accountInfo() {
        System.out.println("Displaying account info");
    }

}

private — fails when called from a different class:

class AccountStatus {

    public static void main(String[] args) {
        Account.accountInfo(); // ERROR: accountInfo() has private access in Account
    }

}

Package declaration and folder structure:

package com.codewala;

class Account {

    public static void main(String[] args) {
        System.out.println("Account class in com.codewala package");
    }

}
// Compile (from the folder containing "com")
javac com/codewala/Account.java

// Run
java com.codewala.Account

Same class name in two different packages (no conflict):

// File: com/codewala/Account.java
package com.codewala;
class Account { }

// File: com/tcs/Account.java
package com.tcs;
class Account { }

default access — accessible within same package:

package com.codewala;

class AccountStatus {

    static void doSomething() { // default access — no modifier
        System.out.println("Do something else");
    }

}

package com.codewala;

class Account {

    public static void main(String[] args) {
        AccountStatus.doSomething(); // works — same package
    }

}

default access — fails from a different package:

package com.tcs;

import com.codewala.AccountStatus;

class AccountManagement {

    public static void main(String[] args) {
        AccountStatus.doSomething();
        // ERROR: doSomething() is not public in com.codewala.AccountStatus;
        // cannot be accessed from outside package
    }

}

❓ Doubts / Questions Raised in Class

QuestionAnswer
What does "private members are not visible outside the class" actually mean?It means another class cannot call/access that member — "visibility" here refers to whether one class can access a member of another class, not literal visual visibility.
Do you need to import a class before using it from another package?Yes — any class not in the current file/package must be imported first, similar to importing a product from another country before using it domestically.
Two classes have the same name in different packages — how do you refer to the correct one?By prefixing the package name before the class name (e.g. com.codewala.Account vs com.tcs.Account) — similar to using a father's name to distinguish two people with the same first name.
If a class is public but a method inside it is private, can the method be called from outside?No — the class being public only controls whether the class itself is accessible from outside the package; each member still follows its own access modifier.

📌 Important Points / Common Mistakes

  • Access modifiers control visibility between classes, not literal code visibility.
  • private = narrowest (class only); default = package only; protected = package + child classes; public = widest (everywhere).
  • Two classes cannot share the same name within the same package/folder — packages solve this by allowing the same class name in different namespaces.
  • Package declaration must be the first line in the Java file, and the physical folder structure must match the package name.
  • A class being public doesn't override the access modifiers of its individual members.
  • Use draw.io to create architecture/flow diagrams — useful for explaining projects in interviews and at work.
