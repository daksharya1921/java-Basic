🧠 Key Concepts

File Name vs Class Name

While compiling, you provide the file name (e.g. account.java). While running, you provide the class name that contains the main method (e.g. AccountInfo).

Multiple Classes → Multiple .class Files

If a single Java file contains multiple classes, compiling it generates one .class file for each class.

Compile vs Build

In industry, developers don't compile files individually using javac. Since a project can have 10,000+ files, a build tool like Maven or Gradle is used to compile the code, manage dependencies, and create JAR files. This process is called a "build", not just "compile".

Reading Compiler Errors

Whenever a compilation error occurs, read the error message carefully — check the file name, line number, and the exact error text. There is no shortcut to this; repeated practice reading errors builds expertise.

What the Compiler Actually Checks

The compiler only checks whether the code follows correct Java syntax — it does not care what the class is named, what the method does, or whether the main method is written correctly. As long as a method declaration is syntactically valid (access modifier + return type + name + body), it will compile — even if it isn't the main method JVM is looking for.

Exact Main Method Signature (Required for Execution) 
public static void main(String[] args) {

}

The code can still compile without this exact signature, but it will not run — JVM specifically looks for a method matching public static void main(String[] args).

Why Main Method is Public

The main method must be callable from outside the class. JVM (which is outside the class) needs to call this method to start execution. If the method were not public, JVM would not be able to call it.

Why Main Method is Static

When the program starts, no object has been created yet. Since static members belong to the class (not an object), JVM can call the main method directly using the class name — ClassName.main() — without needing to create an object first.

Why Main Method is Void

The caller of main is JVM. Since JVM does not use or expect any returned value, there is no purpose in returning anything — hence the return type is void (nothing is returned).

Method Naming Convention — camelCase

The first word starts with a lowercase letter; every subsequent word starts with an uppercase letter (e.g. calculateInterest, doSomething). This is different from class naming, where every word starts with an uppercase letter.

Purpose of String[] args

Used to supply input values to a program at the time of execution via the command line, instead of hardcoding them. These inputs are stored inside a String array. Example: args[0] holds the first value passed on the command line.

Multiple Main Methods in One Class

A class can have multiple overloaded main methods (different parameter lists). The code will compile and run — but JVM will only execute the one matching the exact signature public static void main(String[] args).

Memory Areas (High-Level Only)
  • MetaSpace — holds bytecode/class metadata; static members also reside here
  • Heap — holds objects
  • Stack — holds method call frames (method execution stack) and local variables

📌 Important Points / Common Mistakes

  • Compilation uses the file name; execution uses the class name.
  • Compiler only checks syntax — it does not validate whether your main method signature is correct.
  • Code can compile successfully but still fail to run if the main method signature is wrong.
  • If a file has multiple classes, each class generates its own .class file.
  • Always read compilation error messages fully — check file name, line number, and message.
  • In industry, "build" (via Maven/Gradle) is the term used instead of "compile" for large projects.
  • Access modifier, static, and return type keywords (public, static, void) can be reordered relative to each other, but the return type must always appear immediately before the method name.
