🧠 Key Concepts

if-else — Recap: Decision Making

if-else is used to take a decision based on a business condition — e.g. checking whether Cash on Delivery is available for a given PIN code (blacklisted PIN codes don't get COD).

Why Loops Are Needed

When the same piece of work needs to be repeated multiple times — just with different input each time (e.g. printing a multiplication table, or checking minimum balance for 2,000 customers) — writing the same line of code repeatedly is inefficient. A loop lets you write the logic once and have it execute repeatedly based on a condition.

for Loop Syntax — Three Parts 
for (initialization; condition; update) {
    // body — business logic
}
  • Initialization — runs only once, sets the starting point (e.g. int i = 0)
  • Condition — checked before every iteration; loop continues only while this is true
  • Update — runs after each iteration (commonly increment, e.g. i++, but can also decrement)
for Loop Execution Order

Sequence: Initialization (once) → Check condition → if true, run body → run update → check condition again → repeat, until the condition becomes false, at which point the loop exits.

Loop Starting Point is Flexible

The initialization value doesn't have to be 0 — it can start from any number based on requirement (e.g. starting a multiplication table from 5 instead of 1).

Iterating Over an Array — .length

Every array has a .length property that gives the total number of elements it holds. This is commonly used as the loop's condition when iterating over all elements: for (int index = 0; index < array.length; index++).

Why Array Index Starts at 0

Array indexing starts from 0 (not 1) as a language design convention — starting at a fixed, unambiguous point (zero) avoids the ambiguity that negative indexing would create (there'd be no clear starting boundary).

while Loop Syntax 
while (condition) {
    // body
    // update
}

The condition is checked first; if true, the body executes, then repeats. If the condition is false from the start, the body never executes.

Infinite Loop

If the update step is missing or never makes the condition false, the loop runs forever — this is called an infinite loop and is a common bug to watch for.

do-while Loop 
do {
    // body
} while (condition);

Unlike while, a do-while loop executes its body at least once before checking the condition — even if the condition is false from the start. Useful for scenarios where something must be shown/attempted first (e.g. displaying an ATM menu before checking the entered PIN).

Choosing for vs while

Use for loop when the number of iterations is known/fixed before the loop starts (e.g. a fixed-size array, a fixed count like 10,000 records). Use while loop when the number of iterations is not known in advance — it depends on some condition being met dynamically (e.g. a number-guessing game where the number of attempts varies per user, or reading user input until a certain condition occurs).

💻 Code Snippets / Examples

Basic for loop — print numbers 0 to 9:

class LoopDemo {

    public static void main(String[] args) {
        LoopDemo.printNumbers();
    }

    private static void printNumbers() {
        for (int i = 0; i < 10; i++) {
            System.out.println(i);
        }
    }

}

Multiplication table starting from a custom number:

private static void printTable() {
    for (int i = 5; i <= 10; i++) {
        System.out.println(2 * i);
    }
}

Iterating over command line arguments using a for loop and .length:

class PrintCities {

    public static void main(String[] args) {
        PrintCities.printAll(args);
    }

    private static void printAll(String[] input) {
        for (int index = 0; index < input.length; index++) {
            String city = input[index];
            System.out.println(city);
        }
    }

}

while loop — basic counting:

private static void countUp() {
    int number = 0;

    while (number < 10) {
        System.out.println(number);
        number = number + 1;
    }
}

Infinite loop — missing update (bug example, avoid this):

int number = 0;
while (number < 10) {
    System.out.println(number);
    // missing: number = number + 1;  → this never becomes false → infinite loop
}

do-while loop — executes at least once:

int number = 100;

do {
    System.out.println(number); // executes once even though condition is false
} while (number < 10);

❓ Doubts / Questions Raised in Class

QuestionAnswer
Why doesn't array indexing start from 1?Starting from 0 gives an unambiguous, fixed starting boundary; allowing negative indexing would create ambiguity about where indexing actually begins.
When should for loop be used vs while loop?Use for loop when the number of iterations is known/fixed in advance (e.g. array length); use while loop when it's unknown and depends on a runtime condition (e.g. number of guesses in a game).
What happens if a while loop's update step is missing?The condition never becomes false, causing an infinite loop.
What's the real difference between while and do-while?while checks the condition before executing the body (may never execute); do-while executes the body once first, then checks the condition (always executes at least once).

📌 Important Points / Common Mistakes

  • for loop initialization runs only once — not on every iteration.
  • Always ensure the update step in a while loop actually moves the condition toward becoming false, to avoid infinite loops.
  • Use .length to iterate arrays dynamically instead of hardcoding array size.
  • Array indexing always starts from 0; the valid index range is 0 to (length - 1).
  • do-while guarantees at least one execution of the loop body, regardless of the condition — useful for "show first, then check" scenarios.
  • Choose the loop type based on whether the number of iterations is known beforehand (for) or not (while).
