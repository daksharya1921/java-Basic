Class 08 — Local, Instance & Static Variables

🎯 Topics Covered

  • Types of variables in Java: Local, Instance, Static
  • Local variables — declared inside a method, scope limited to that method
  • Instance variables — declared inside a class, outside methods; belong to the object
  • Static variables — belong to the class; one copy shared across all objects
  • Memory location for each variable type (Stack vs Heap vs MetaSpace)
  • Accessing static variables using ClassName.variableName
  • When to use static vs instance (e.g. GST/interest rate example)
  • Practice: calculating total amount using local and static variables together

🧠 Key Concepts

Local Variable

A variable declared inside a method (or block). Its scope is limited to that method only — it cannot be accessed from another method. Trying to access a local variable outside the method where it's declared causes a compilation error: cannot find symbol.

Instance Variable

A variable declared inside the class but outside any method. It belongs to the object (also called an "instance"). Every object created from the class gets its own separate copy of the instance variable. Example: in a banking application, "balance" is an instance variable — each user (object) has their own balance value, but they all share the same variable definition (e.g. balance) in the class.

Static Variable

A variable declared with the static keyword. It belongs to the class, not to any individual object — there is only one copy shared across all objects of that class. Used when a value is the same for every object — e.g. an interest rate or GST percentage that applies equally to all customers/products, so there's no need to store a separate copy per object.

Why Use Static Instead of Instance (Example: GST)

If GST (18%) were declared as a non-static (instance) variable, then calculating GST for 1 lakh customers would create 1 lakh separate copies of the same value — wasting memory. Since the value is identical for everyone, it should be static, so only one copy exists and is shared by all.

Accessing Static Variables

Static members are accessed using the class name, not an object: ClassName.variableName. This is the same reason the main method can be called by JVM without creating an object — because main is static.

Memory Location of Each Variable Type
  • Local variable → stored in the Stack
  • Instance variable → stored in the Heap (since it belongs to the object, and objects live in the heap)
  • Static variable → stored in the Heap (prior to Java 8 it was stored in PermGen/MetaSpace; from Java 8 onward, static variables are part of the Heap)
  • Class metadata/bytecode → stored in MetaSpace (also called the method area)
Declaration vs Assignment

Declaration = specifying the data type and name of a variable. Assignment = giving that variable a value using the = operator. These can happen together (int balance = 2000;) or separately (declare first, assign later).

💻 Code Snippets / Examples

Local variable — only accessible within its own method:

class Variables {

    public static void main(String[] args) {
        int balance = 2000; // local variable — scope limited to main()
        System.out.println(balance);
    }

    public static void view() {
        System.out.println(balance); // ERROR: cannot find symbol — balance not visible here
    }

}

Instance variable — accessible by any method in the class:

class Variables {

    int amount = 2000; // instance variable — belongs to the object

    public static void main(String[] args) {
        // instance variable cannot be accessed directly from a static method
        // without an object reference (covered when objects are introduced)
    }

}

Static variable — shared across all objects, accessed via class name:

class Variables {

    static int interestRate = 5;

    public static void main(String[] args) {
        System.out.println(Variables.interestRate); // accessed using ClassName.variableName
    }

}

Practice exercise — total amount using local + static variables:

class ProductTotal {

    static int gst = 18; // static — same GST slab applies to all products in this range

    public static void main(String[] args) {
        String productName = args[0];       // local
        int price = Integer.parseInt(args[1]); // local

        int gstAmount = (price * gst) / 100;
        int totalPrice = price + gstAmount;

        System.out.println("Product: " + productName);
        System.out.println("Total Price: " + totalPrice);
    }

}

📌 Important Points / Common Mistakes

  • Local variables cannot be accessed outside the method they're declared in — this is a common source of "cannot find symbol" compiler errors.
  • Instance variables belong to the object, not the class — each object gets its own copy.
  • Static variables belong to the class — only one shared copy exists for all objects.
  • Use static for values that are genuinely the same across all objects (e.g. tax rates, slabs); use instance for values that vary per object (e.g. individual account balance).
  • Always compile using the full file name with the .java extension; run using only the class name (no extension).
  • Hardcoded GST/tax values inside logic should instead use a properly typed static variable, not a string or hardcoded literal.
