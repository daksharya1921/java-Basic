🧠 Key Concepts

What is a Variable?

A variable is a container that stores data/value. It is called a "variable" because its value may change over time (e.g. an amount can be 100 today and 200 later).

Variable Declaration Syntax 
dataType variableName = value;

The = here is not mathematical equality — it is the assignment operator, used to assign a value to a variable. Every variable must be given a data type before it can be declared.

Primitive vs Non-Primitive Data Types

Java has two categories of data types:

  • Primitive data types — fixed, built-in types: byte, short, int, long, float, double, char, boolean. These are not objects — this is one reason Java is not 100% object-oriented.
  • Non-primitive data types — any object type (e.g. String, or a custom class like Employee, Account, Payment). The number of non-primitive types is unlimited, since developers can create their own classes.
Why Multiple Integer Types Exist (byte, short, int, long)

All of these store integers, but they differ in memory size, which determines their value range:

  • byte → 8 bits → range: -128 to 127
  • short → 16 bits → range: -32,768 to 32,767 (2^16 based)
  • int → 32 bits
  • long → 64 bits (largest range among integer types)

The range is calculated using the formula based on 2^(number of bits), split between negative and positive values (0 is included on the positive side).

Decimal Data Types
  • float → 32 bits — lower range/precision for decimals
  • double → 64 bits — larger range, used when more precision/range is needed for decimal values

Instructor noted float is the "lowest" for decimals and double is used for a bigger range, mirroring how byte is lowest and long is highest among integer types.

Boolean and Char
  • boolean → stores only true or false (e.g. used for a flag like "is work done or not")
  • char → stores a single character (e.g. 'A')
Choosing the Right Data Type

Before writing code, decide the data type based on the actual requirement/attribute. Example: for employee details — name/address would be String, age could be int/short/byte, salary could be int/double, and a "work done" flag would be boolean.

Out-of-Range Values Fail to Compile

If a value assigned to a variable exceeds its data type's range (e.g. assigning 128 or 130 to a byte, whose max is 127), the compiler throws an error: incompatible types: possible lossy conversion from int to byte. In such cases, choose a larger type (e.g. short or int).

Concatenating Static Text with Variable Values

To print a mix of fixed text (static) and a variable's value (dynamic), use the + operator to join them.

💻 Code Snippets / Examples

int amount = 100;
amount = 200; // value changes — this is why it's called a "variable"
boolean status = true;
class Age {

    public static void main(String[] args) {
        byte age = 18;
        System.out.println("Age is " + age + " years");
    }

}

Out-of-range example (fails to compile):

byte age = 130; // ERROR: incompatible types, possible lossy conversion from int to byte
// byte range is -128 to 127

Using short instead when byte's range is insufficient:

short amount = 5000;

Float and double for decimals:

float temperature = 98.6f; // 'f' suffix required for float
double piValue = 3.14159; // larger range/precision

Char example:

char grade = 'A';

📌 Important Points / Common Mistakes

  • Decide the data type based on the actual requirement before writing code.
  • Variable naming must follow camelCase — first word lowercase, each subsequent word capitalized (e.g. orderId, not OrderID or order_ID).
  • Do not add unnecessary underscores in variable names unless genuinely required.
  • String values must always be in double quotes; an unclosed quote causes an "unclosed character literal" compiler error.
  • Use boolean (not String) for true/false flags — e.g. isOpenBoxDelivery should be boolean, not a string like "true"/"yes".
  • Class names must start with a capital letter (e.g. OrderDetails, not orderDetails).
  • Naming convention mistakes are treated very seriously — instructor emphasized that incorrect capitalization (e.g. a variable starting with a capital letter) is an immediate red flag in real code review/interviews.
