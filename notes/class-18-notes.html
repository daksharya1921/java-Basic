🧠 Key Concepts (From Available Fragments)

Every Class Implicitly Extends Object

In Java, every class you create is automatically a child of the built-in Object class, even if you don't write extends Object explicitly. Because of this, every Java class automatically gets certain methods that "every Java class requires" — the instructor mentioned equals and hashCode as examples of such inherited methods.

Why a Class Can Call a Method It Never Defined

If you create an object of your own class (e.g. P1) and can call a method like hashCode() that isn't written anywhere in your class, it's because that method comes from the parent Object class — inherited automatically.

Parent Must Be Initialized Before Child

Before a child class object can be created/used, its parent class must be initialized first. This is why access to inherited members (like Object class methods) works — the parent's initialization happens as part of creating the child object.
